# Operations Documents

This layer is reserved for repeatable run procedures and practical project operation.

Expected documents:

- `commands.md` for dataset preparation, graph construction, retrieval, tuning, evaluation, and aggregation commands.
- `logging.md` for console output, structured run metadata, and debug artifact conventions.
- `reproducibility.md` for seeds, config precedence, run summaries, and experiment traceability.
- `debug-artifacts.md` for graph stats, score breakdowns, failure cases, and leakage checks.
- `implementation-handoff.md` for post-implementation code review entry points, control flow, key abstractions, and review checklist.
- `troubleshooting.md` for common failure modes and how to diagnose them.

Keep this layer focused on how to run and inspect the system, not on research motivation or schema definitions.
