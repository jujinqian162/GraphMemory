# Design Documents

This layer is reserved for durable design decisions that sit below project requirements and above implementation details.

Expected documents:

- `architecture.md` for module layering and data flow.
- `abstractions.md` for retriever, graph builder, reranker, evaluator, and validator interfaces.
- `naming-conventions.md` for module, function, class, variable, artifact, method, and graph naming.
- `testing-strategy.md` for test layers, fixtures, network/model policy, and verification tiers.
- `validation-strategy.md` for fail-fast artifact validation and error style.

Move decisions here after they have stabilized in brainstorm or planning notes.
